# TodoList Demo — 低保真原型文档

版本：1.0
日期：2025-12-04

目的：为 `todoList_demo` 产出清晰的信息架构、关键页面线框（低保真）、交互说明与验收点，作为前端实现的直接参考。

一、关键用户流程

1. 新建任务
   - 用户点击顶部“新建任务”输入框或按钮
   - 在行内输入标题（必填），可选择填写描述/到期日/优先级/标签
   - 点击“添加”后任务出现在任务列表顶部（或按排序规则）

2. 编辑任务
   - 点击任务标题打开编辑面板（右侧抽屉或模态）
   - 修改字段并保存，关闭编辑面板后列表显示更新

3. 标记完成/取消完成
   - 点击任务左侧复选框切换完成状态
   - 完成的任务显示为灰色并带删除线或完成标签

4. 搜索/筛选/排序
   - 在搜索框输入关键字，列表即时过滤
   - 筛选按钮：全部 / 未完成 / 已完成
   - 排序下拉：创建时间（默认）、到期日、优先级

5. 删除与撤销
   - 删除后显示顶部/底部 Toast：已删除，撤销（5 秒）

二、页面与组件清单

- 页面
  - 任务列表页（主页面）
  - 任务详情/编辑抽屉（或模态）
  - 设置页（非 MVP）

- 复用组件
  - `Header`：包含应用名、搜索框、用户入口（后续）
  - `FilterBar`：筛选/排序/新建入口
  - `TaskList`：渲染 `TaskItem` 列表
  - `TaskItem`：复选框、标题、优先级、到期日、操作按钮（编辑/删除）
  - `TaskForm`：用于创建/编辑任务（可作为抽屉或模态）
  - `Toast`：操作反馈与撤销

四、交互与可访问性要点

- 所有可点击元素要有明确的焦点样式（键盘可操作）
- 复选/按钮需支持触控（移动设备）
- 表单校验：标题必填；输入过长显示最多 200 字
- 颜色对比：保证文本与背景对比满足 AA

五、验收标准（与 PRD 对应）
- 主流程（创建->标记完成）在 3 次交互内完成
- 搜索/筛选能即时生效且在 200ms 内完成（取决实现）
- 编辑保存后任务数据立刻反映在主列表
- 删除后可在 5 秒内撤销

六、实现建议（前端）
- 使用 `localStorage` 作为原型数据持久化；联调阶段替换为 API
- 使用 CSS 变量控制主题色与间距；Tailwind 可快速实现
- 将 `TaskForm` 作为受控组件，使用 `React Hook Form`（若使用 React）
